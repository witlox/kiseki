# RFC 4506 §4.10 — Variable-Length Opaque Data fixture

## Source
RFC 4506 §4.10 (XDR: External Data Representation, May 2006):
  https://www.rfc-editor.org/rfc/rfc4506#section-4.10

The section text walks through the encoding of an opaque payload of
5 bytes. The fixture in this directory IS that example, byte-for-byte:

  length prefix : 00 00 00 05  (uint32 = 5)
  payload       : 01 02 03 04 05
  pad           : 00 00 00     (round to 12 = 4*3 boundary)
  total         : 12 bytes

## Cross-impl notes
Any compliant XDR encoder MUST produce these exact bytes for the
input `opaque<5> = {1,2,3,4,5}`. kiseki's `XdrWriter::write_opaque`
is verified against this fixture in
`crates/kiseki-gateway/tests/rfc4506.rs::s4_10_opaque_matches_vendored_fixture`.
